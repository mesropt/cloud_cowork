# Research Synthesizer Skill

## Когда использовать

- Анализ 10+ документов любого типа (статьи, заметки, транскрипты, отчёты)
- Сравнение нескольких источников с поиском паттернов и противоречий
- Комбинирование внутренних данных с актуальной информацией из интернета
- Подготовка стратегических брифов на основе большого объёма материала

## Входные данные

- Папка с документами (укажи путь или название)
- Опциональный фокусный вопрос: «Что делает X успешным?», «Каковы главные риски?»
- Опциональный запрос на веб-поиск: тема, конкурент, рыночный контекст

## Формат вывода

Отчёт всегда включает следующие разделы:

1. **Executive Summary** — 3–5 предложений, главный вывод
2. **Ключевые темы** — 2–4 доминирующих паттерна с источниками
3. **Данные и статистика** — конкретные цифры, сгруппированные по категориям
4. **Противоречия и напряжения** — где источники расходятся и почему это важно
5. **Стратегические рекомендации** — действия, вытекающие из анализа
6. **Вопросы для дальнейшего исследования** — что осталось неясным

## Когда запрашивать суб-агенты

Явно запроси параллельную обработку если:
- Документов больше 20
- Нужно проанализировать разные категории независимо (например: статьи отдельно, встречи отдельно)
- Нужна многоперспективная оценка одного решения (финансовая / операционная / клиентская)

Пример делегации: *«Проанализируй эти 50 документов с использованием суб-агентов. Найди главные темы, противоречия и дай стратегические рекомендации.»*

## Как подключить веб-поиск

Добавь к делегации: *«После анализа локальных файлов найди актуальный контекст по [теме/конкуренту] в интернете и синтезируй с локальными данными.»*

**Правило источников:**
- Локальные файлы → проприетарные данные, история, внутренний контекст
- Веб-поиск → текущие события, конкуренты, рыночные тренды, публичные данные
- Комбинация → самый мощный формат для стратегического анализа
